# Mission — justerm 코드베이스를 지휘·검수할 수 있게

## 한 줄
justerm 을 Claude Code 에이전트로 만들면서, **뭐가 만들어지는지 실제로 이해한 채** 방향을 잡고
산출물을 리뷰(검수)할 수 있게 된다.

## 역할
- 사용자는 **코드를 직접 짜는 사람이 아니라 지휘·검수하는 사람**이다.
  → 목표는 "Rust 를 손으로 짤 수 있게"가 아니라, **읽고 이해해서 에이전트 산출물을 리뷰하고
     방향을 잡을** 만큼의 멘탈 모델.
- 따라서 레슨은 Rust 문법 mechanics 를 깊게 파지 않는다. 대신:
  - 이 엔진의 **정체성/경계**(무엇을 하고 무엇을 안 하는가)
  - **핵심 모델**(cell·damage·viewport·scrollback·selection·cadence)
  - **설계 결정의 '왜'**(ADR)
  - 에이전트가 자주 저지르는 **위반 패턴**을 리뷰에서 잡아내는 법

## 왜 지금
다른 작업환경에서 clone 해와서 개인 메모리가 비어있는 상태. 정체성/계약/flow 는 repo 에 다 있지만
(CLAUDE.md·docs/architecture.md·docs/adr/), 사용자 본인이 그 내용을 **내재화**하지 못해 에이전트에
의존적. 이 갭을 메우는 것.

## 성공의 모습 (관측 가능한 지표)
1. 에이전트가 제안한 변경을 보고 "이거 경계 위반 아니야?"를 **스스로** 짚는다.
2. ADR 을 근거로 "왜 이렇게 했지?"에 답할 수 있다.
3. damage/cadence/cell 같은 핵심 용어를 정확히 쓴다.
4. core 냐 소비처냐 라우팅(ADR-0017)을 판단할 수 있다.

## 현재 baseline (2026-07-11)
- Rust: 거의 처음
- 터미널/VT 도메인: 처음
- justerm 코드베이스: 거의 모름 (에이전트로 빌드는 해왔음)
