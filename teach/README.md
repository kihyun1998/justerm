# teach/ — 개인 `/teach` 코스 워크스페이스

이 폴더는 사용자가 `/teach` 스킬로 **justerm 코드베이스를 직접 이해**하려고 진행 중인
다세션 학습 코스의 **산출물이자 정본(canonical)** 이다. (스킬 본체는 각 머신의 유저홈
`~/.claude/skills/teach/` 에 있고 repo 엔 없다 — 산출물만 여기 둔다.)

## 에이전트 규칙 (다른 환경에서 `/teach` 이어갈 때)

- **워크스페이스 = 이 `teach/` 폴더.** `/teach` 스킬은 "현재 디렉터리를 워크스페이스로 취급"
  하므로, **반드시 이 폴더 안에서** `/teach` 를 실행/작성한다. 새 레슨·학습기록은 전부 여기 쓴다.
- **쓰면 커밋한다.** 이 자료는 repo 에 커밋돼야 다른 환경에 동기화된다(원래 `~/.claude` 밖 사본은
  더 이상 정본 아님). 레슨·record 추가/수정 후 커밋 필수.
- **진행 이어가기**: `learning-records/` 의 **최신 번호부터** 읽어 ZPD(현재 수준·다음 할 것)를 파악한다.
- 새 트랙 진입 시 짧은 배치질문으로 ZPD 재보정(사용자가 이 방식을 선호).

## 구성

- `MISSION.md` — 코스 목표(왜 배우나). 모든 교육 결정의 근거.
- `lessons/*.html` — 레슨 본문(1주제 1파일, `assets/course.css` 링크).
- `learning-records/*.md` — 세션별 진행·피벗·배치고사 기록(진도의 소스오브트루스).
- `reference/curriculum-map.html` — 4트랙 커리큘럼 지도, "너 여기" 마커.
- `reference/glossary.html` — 워크스페이스 공용 용어.
- `NOTES.md` · `RESOURCES.md` · `assets/`.

## 진행 상황 (최근 기록 기준 요약 — 정확한 최신은 항상 `learning-records/` 로 확인)

- **Mission**: 에이전트 작업을 **지휘·검수**할 수 있게 되기. 코드를 직접 짜는 게 아니라 읽고 이해해
  리뷰/방향잡기 → 레슨은 Rust 문법보다 **경계·모델·ADR '왜'·위반 패턴** 중심.
- **Baseline**: Rust 거의 처음, VT 도메인 처음, justerm 코드 거의 모름(에이전트로 빌드만 해옴).
- **레슨 1**("경계 = 정체성") 완료 후, 사용자 지적 반영해 통째 재작성: "justerm = 풀스택 패밀리,
  경계는 크레이트마다 다름, 리뷰 첫 질문 = 어느 크레이트냐"가 골격.
- **배치고사**: 사용자 프런티어 = 기초(터미널·이스케이프·파서·폭이 아직 어렴풋).
- **커리큘럼 척추 순서**: 1→2→3→4→5→8→(9·10 봄)→11→12. 숨은 VT 상태·selection·ADR 지도는 dip-in 참조.
