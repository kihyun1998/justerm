# Teaching notes / 사용자 선호

## 언어
- 사람이 읽는 문서·레슨·대화: **한국어**. (repo 규칙과 일치: 코드 주석/CONTEXT/ADR 만 영어)

## 수준
- Rust 거의 처음 · VT 도메인 처음. → 코드 스니펫을 보여줄 땐 반드시 **한 줄씩 풀어서** 설명.
  Rust 문법(소유권·라이프타임·트레잇)은 "몰라도 리뷰되는" 선에서만, 필요할 때 곁들인다.
- 목표는 **리뷰어/디렉터** 관점. "이게 왜 여기 있나 / 이게 경계를 넘나"를 판단하는 힘 위주.

## 스타일
- 레슨은 짧게, 한 번에 win 하나. 워킹 메모리 존중.
- 실제 repo(CLAUDE.md·architecture.md·ADR·이슈)를 1차 출처로 삼고 인용. 파라메트릭 지식 불신.
- 각 레슨 끝에 retrieval 퀴즈(즉시 피드백) + "모르면 선생(에이전트)한테 물어봐" 리마인더.
