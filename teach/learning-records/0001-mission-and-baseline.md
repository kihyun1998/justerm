# 0001 — Mission 확정 + baseline + 코스 방향

날짜: 2026-07-11

## 상황
사용자가 다른 작업환경에서 justerm 을 clone 해와 개인 메모리가 빈 상태. 에이전트(Claude Code)로
justerm 을 빌드해왔지만 코드베이스를 본인이 내재화하지 못해 "잘 할 수 있을지 모르겠다"며 `/teach` 시작.

## 확정된 것 (사용자 응답)
- **Mission**: 에이전트 작업 **지휘·검수**. 코드를 직접 짜기보다, 뭐가 만들어지는지 이해한 채 방향 잡고 리뷰.
- **Rust**: 거의 처음.
- **VT 도메인**: 처음.

## 이게 코스에 주는 함의 (비-자명한 결정)
1. **Rust 문법을 코스의 중심에 두지 않는다.** 목표가 작성이 아니라 리뷰라서. 소유권/트레잇/라이프타임은
   "리뷰에 걸릴 때만" 곁들인다. → 대신 **경계·모델·ADR '왜'·위반 패턴**이 중심.
2. **첫 레슨 = 경계 invariant.** 이유: 리뷰어에게 가장 레버리지 큰 무기. CLAUDE.md 가 스스로 "이게 정체성"
   이라 부르고, 터미널 엔진에서 가장 흔한 실수 종류(경계 위반·우회)를 통째로 잡는 체크리스트를 준다.
   추상 이론(cell 내부구조 등)보다 이게 ZPD 에 맞고 즉시 실전(PR 리뷰)에 쓰인다.
3. **repo 자체가 최상위 교재.** 파라메트릭 지식 대신 CLAUDE.md/architecture.md/ADR/이슈 인용.
